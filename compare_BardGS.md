# BARD-GS 与 MonoGS Baseline 对比分析

## 1. 一句话结论

BARD-GS 不是 SLAM 系统，而是一个面向“动态场景 + 运动模糊图像”的离线 3DGS 重建方法。它最值得借鉴的点是：不要先把模糊图像当成清晰图像去优化，而是显式模拟曝光时间内的相机运动和物体运动，再把多个瞬时渲染结果平均成一张模糊图，与输入图像对齐。

对 MonoGS 这类 Gaussian Splatting SLAM 来说，BARD-GS 中“相机运动模糊建模”最可能提升跟踪 ATE RMSE；“动态物体掩码、动态高斯轨迹约束、深度/光流先验”最可能提升建图 PSNR、SSIM、LPIPS，但完整移植动态高斯形变场的工程代价较高。

## 2. BARD-GS 的基本信息

| 项目 | 内容 |
|---|---|
| 论文 | BARD-GS: Blur-Aware Reconstruction of Dynamic Scenes via Gaussian Splatting |
| 任务 | 从有运动模糊的单目 RGB 视频/图像中重建动态 3D 场景 |
| 输入 | 模糊 RGB 图像、COLMAP 初始相机位姿、动态区域 mask、2D 轨迹、单目深度先验 |
| 是否 RGB-D | 不是 RGB-D；深度来自 Depth-Anything 这类单目深度网络，属于伪深度先验 |
| 是否在线 SLAM | 不是；它是离线优化式重建 |
| 主要指标 | PSNR、SSIM、LPIPS，额外使用 LV 和 MUSIQ 衡量清晰度 |
| 是否评估 ATE | 没有把 ATE 作为核心指标；相机位姿主要用于重建和新视角合成 |

## 3. BARD-GS 的跟踪/位姿策略

BARD-GS 没有 MonoGS 意义上的在线 tracking 前端。它的“位姿相关策略”是相机运动模糊建模：

1. 先用 COLMAP 得到每张模糊图像的初始相机位姿。
2. 对每张图像，不只优化一个相机位姿，而是优化曝光开始和曝光结束两个虚拟位姿。
3. 在这两个虚拟位姿之间插值，得到一组曝光时间内的虚拟相机。
4. 用这些虚拟相机分别渲染多张瞬时清晰图，再求平均，合成一张“预测模糊图”。
5. 将预测模糊图与真实输入模糊图做光度损失。
6. 第一阶段只用静态区域优化相机轨迹，动态区域由 Track-Anything 生成的 mask 排除，避免运动物体干扰位姿。
7. 推理/评估时，用曝光中点位姿作为优化后的相机位姿。

通俗理解：普通 3DGS 假设一张照片对应一个瞬间；BARD-GS 认为模糊照片是曝光时间内很多瞬间的平均结果。因此它优化的是“一小段相机运动轨迹”，不是单个静止相机。

## 4. BARD-GS 的建图策略

BARD-GS 的建图分两阶段：

### 4.1 阶段一：相机运动去模糊，重建静态区域

这一阶段只优化静态区域。动态物体区域被 mask 掉，因为动态物体本身也会产生模糊，如果让它参与位姿优化，会把相机运动和物体运动混在一起。

核心目标是：用虚拟相机轨迹解释静态背景的模糊，从而得到更清晰的静态高斯和更可靠的相机位姿。

### 4.2 阶段二：物体运动去模糊，重建动态区域

第二阶段引入时间条件形变场：

```text
F(x, t) -> delta_position, delta_rotation, delta_scale
```

意思是：同一个 3D Gaussian 在不同时间可以移动、旋转、缩放。这样就能表达动态物体在曝光时间内的位置变化。

具体做法：

1. 对每个输入时间，采样多个虚拟时间点。
2. 用形变场得到这些时间点上的动态高斯位置。
3. 对每个虚拟时间点渲染一张瞬时图像。
4. 把多张瞬时图像平均成预测模糊图。
5. 与真实模糊图做整图渲染损失。
6. 使用 BootsTAPIR 得到 2D 像素轨迹，结合 Depth-Anything 深度和相机位姿反投影到 3D，形成伪 3D 轨迹监督。
7. 初期用 3D track loss 约束动态高斯跟随真实物体运动；之后去掉该约束，继续优化渲染质量。

通俗理解：如果一辆玩具车在曝光期间移动，BARD-GS 不会强行让一堆静止高斯去拟合拖影，而是让高斯沿着车的运动轨迹移动，再把不同时间的渲染平均起来形成拖影。

## 5. 使用的数据集

| 数据集 | 用途 | 构造方式 | 说明 |
|---|---|---|---|
| HyperNeRF 子集 | 合成模糊去模糊评估 | 用 RIFE 插帧提高帧率，再平均连续清晰帧生成模糊图 | 选了 3 个场景 |
| DyCheck 子集 | 合成模糊去模糊评估 | 同上 | 选了 4 个场景 |
| BARD-GS Real-world Blurry Dataset | 真实模糊新视角合成评估 | 双 GoPro 同步采集：1/24s 曝光采集模糊训练图，1/240s 曝光采集清晰评估图 | 共 12 个动态场景 |

注意：BARD-GS 的真实数据是单目 RGB 模糊输入，不是 RGB-D。它需要外部工具提供辅助信息，例如 COLMAP 位姿、Track-Anything 动态 mask、BootsTAPIR 轨迹和 Depth-Anything 深度。

## 6. 与 MonoGS Baseline 的差异

| 维度 | MonoGS Baseline | BARD-GS |
|---|---|---|
| 系统类型 | 在线 SLAM | 离线动态场景重建 |
| 输入 | 支持 monocular / RGB-D / stereo / RealSense | 单目 RGB 模糊图像 |
| 跟踪 | 前端固定高斯，优化当前相机位姿，最小化渲染图与输入图的误差 | 从 COLMAP 位姿出发，优化曝光时间内的虚拟相机轨迹 |
| 建图 | 后端局部 BA，优化滑窗关键帧位姿和高斯参数，并做 densification / pruning | 两阶段优化：先静态去相机模糊，再动态去物体模糊 |
| 动态物体处理 | baseline 没有显式动态物体形变建模 | 使用动态 mask、形变场和 3D track loss |
| 模糊处理 | 默认把输入当作清晰瞬时图像 | 显式模拟运动模糊形成过程 |
| 速度 | 面向实时 | 80,000 次迭代，明显偏离线 |

## 7. 能否嵌入 MonoGS 来提升指标

可以，但建议分层移植，不能直接照搬完整 BARD-GS。原因是 MonoGS 的目标是实时 SLAM，而 BARD-GS 是离线重建；完整引入动态形变场和多虚拟视角渲染会显著增加计算量。

### 7.1 最推荐：相机运动模糊感知跟踪

迁移位置：`utils/slam_frontend.py` 的 tracking 过程，以及 `utils/slam_utils.py` 的 tracking loss。

改进思路：

1. 当前 MonoGS 每帧只优化一个相机位姿。
2. 可以改成优化一个中心位姿加一个曝光运动增量。
3. 在曝光时间内采样少量虚拟位姿，例如 3 或 5 个。
4. 多次渲染后平均，得到预测模糊图。
5. 用预测模糊图与输入图计算 tracking loss。
6. 对疑似动态区域、低置信区域或高残差区域降权。

预期收益：

| 指标 | 可能影响 |
|---|---|
| ATE RMSE | 在相机抖动、低光照、快速运动导致模糊时，可能下降 |
| PSNR/SSIM/LPIPS | 位姿更准后，建图对齐更好，间接提升渲染质量 |

风险：

1. 每帧渲染次数增加，实时性下降。
2. 如果输入本身清晰，虚拟位姿可能引入不必要自由度。
3. 动态 mask 错误时，可能排除过多有效区域。

建议实现为可开关模块，例如 `blur_aware_tracking.enabled`，只在检测到模糊或快速运动时启用。

### 7.2 推荐：动态区域 mask 降权

迁移位置：`utils/slam_utils.py` 的 tracking 和 mapping loss。

BARD-GS 第一阶段明确排除动态区域，这一点非常适合 SLAM。对 SLAM 来说，动态物体会破坏位姿估计，因为它们不满足“场景静止”的基本假设。

改进思路：

1. 用 Track-Anything、SAM2、光流一致性或现有不确定性网络得到动态/不可靠区域。
2. tracking 时对这些区域降权或直接 mask。
3. mapping 时不要让动态区域主导高斯优化和致密化。

预期收益：

| 指标 | 可能影响 |
|---|---|
| ATE RMSE | 动态物体干扰减少，位姿更稳 |
| PSNR/SSIM/LPIPS | 背景地图更干净，动态拖影和错误高斯减少 |

这是最现实、收益风险比最高的移植点。

### 7.3 中等难度：模糊感知建图损失

迁移位置：`utils/slam_backend.py` 的 `map()` 和 `utils/slam_utils.py` 的 mapping loss。

改进思路：

1. 对关键帧建图时，不再只用单次渲染图对齐输入。
2. 对疑似模糊关键帧，渲染多个虚拟位姿并平均。
3. 用平均图与输入图计算 RGB loss / SSIM loss。
4. 清晰关键帧仍使用原始 MonoGS loss，避免额外开销。

预期收益：

| 指标 | 可能影响 |
|---|---|
| PSNR | 模糊帧不再强迫高斯学成拖影，可能提升 |
| SSIM | 结构更清晰，可能提升 |
| LPIPS | 动态和模糊区域的感知质量可能改善 |

风险是后端每次优化需要多次渲染，训练成本增加。

### 7.4 高难度：动态高斯形变场

迁移位置：需要新增动态高斯表示、形变网络、时间输入、动态 mask、轨迹监督和渲染接口，改动范围会覆盖 GaussianModel、renderer、backend 和 dataset。

预期收益：

| 指标 | 可能影响 |
|---|---|
| PSNR/SSIM/LPIPS | 在动态场景中提升明显 |
| ATE RMSE | 不一定直接提升，除非动态区域同时被用于更好的 mask 或运动解释 |

不建议作为第一阶段工作。它更适合作为一篇完整论文方法的主体，而不是 baseline 的小改动。

## 8. 建议的论文创新路线

如果论文 baseline 是 Gaussian Splatting SLAM，建议不要把目标写成“完整复现 BARD-GS”，而应写成：

> 面向运动模糊和动态干扰的 Blur-aware Gaussian Splatting SLAM。

可以设计三个层级模块：

| 模块 | 核心思想 | 主要影响指标 | 实现难度 |
|---|---|---|---|
| Blur-aware Tracking | 用虚拟相机轨迹解释模糊帧，而不是用单位姿硬拟合 | ATE RMSE | 中 |
| Dynamic/Uncertain Masking | 动态物体和不可靠区域不参与或少参与位姿优化 | ATE RMSE、PSNR、SSIM、LPIPS | 低到中 |
| Blur-aware Mapping | 对模糊关键帧使用多虚拟位姿平均渲染损失 | PSNR、SSIM、LPIPS | 中 |
| Dynamic Gaussian Motion | 对动态物体单独建模形变轨迹 | PSNR、SSIM、LPIPS | 高 |

最稳妥的实验路径：

1. 先加动态/不确定区域 mask，验证 ATE RMSE 是否下降。
2. 再加 blur-aware tracking，验证模糊序列上 ATE RMSE 是否进一步下降。
3. 最后把 blur-aware loss 放入 mapping，验证 PSNR、SSIM、LPIPS。
4. 若前三步有效，再考虑动态高斯形变场。

## 9. 对指标的具体预期

| 指标 | BARD-GS 思想可能带来的变化 | 原因 |
|---|---|---|
| ATE RMSE | 模糊/动态场景中可能下降 | 位姿优化不再被模糊图和动态物体强干扰 |
| PSNR | 模糊关键帧上可能提升 | 高斯不会被迫拟合错误拖影 |
| SSIM | 可能提升 | 静态结构和动态边界更清楚 |
| LPIPS | 可能下降 | 动态区域的感知质量更自然 |

但如果测试数据主要是静态、清晰、RGB-D 数据，BARD-GS 模块的收益可能不明显，甚至因为额外自由度导致轻微退化。因此实验应至少包含快速相机运动、低光照模糊或动态物体场景。

## 10. 最终建议

BARD-GS 对 MonoGS 最有价值的不是“动态重建框架本身”，而是三个思想：

1. 模糊图像是曝光时间内多个瞬时图像的平均，不应当用单一位姿直接拟合。
2. 位姿优化时应优先相信静态区域，动态区域需要 mask 或降权。
3. 动态物体的高斯应该有运动轨迹约束，否则容易生成拖影和错误几何。

因此，建议你的论文方法以“Blur-aware Tracking + Dynamic-aware Mapping”为主线。第一阶段做轻量级、可实时化的相机运动模糊建模和动态区域降权，主攻 ATE RMSE；第二阶段做关键帧建图中的模糊感知渲染损失，主攻 PSNR、SSIM、LPIPS。这样既能吸收 BARD-GS 的核心优点，又不会破坏 MonoGS 作为 SLAM 系统的在线结构。
